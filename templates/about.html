About AutoTCGA

AutoTCGA implements a machine learning pipeline that pre-processes biological data and uses it to train classifiers to predict immune-evasion events. The tool allows the full personalization of the dataset, from a selection of available sources with different attributes, and any attribute (not just the immune-evasion events)can be selected as class (using the 'Other' option), while the remaining will be used as training features. The models are optimized with a parameter grid approach, using RandomSerchCV and GridSearchCV algorithms from scikit-learn python library.

Instructions:

  • Algorithm: tree-based classifier chosen to be trained with dataset after pre-processing and personalization.
  • Class: attribute chosen to be used as class in prediction.
  • Dataset(s): source of features used to train the algorithm (the chosen class must belong to this dataset or to the attributes given as input).
  • Population: cancer type of the patients, works as row filter.
  • Pre-processing: mode defined by a number between 1 and 6 which represent a combination of three techniques (normalization, categorization and creation of dummy variables).
  • Feature Selection and Feature Engineering: set of feature techniques to be applied to the dataset after pre-processing and personalization. Feature Engineering is only available for the dataset of PanImmune source.
  • Feature Selection and Feature Engineering: set of feature techniques to be applied to the dataset after pre-processing and personalization. Feature Engineering is only available for the dataset of PanImmune source.
  • Sub-set of features to exclude: features from the chosen dataset(s) that the user wishes to exclude; all other will remain.
  • Sub-set of features to include: features from the chosen dataset(s) that the user wishes to remain; all others wil be excluded.
  • User input columns: new attributes added from the user, including a mandatory 'sample_code' column with the TCGA code in the correct data shape.
